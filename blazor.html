Clases y Objetos

En la programación orientada a objetos (POO), una **clase** es como un plano o un molde para crear objetos. Define las propiedades (campos) y las acciones (métodos) que un objeto tendrá.

Un **objeto** es una instancia de una clase. Es el producto final creado a partir del plano, con sus propios valores para las propiedades definidas.

Clase (Plano)
public class Coche {
    public string Color;
    public int Velocidad;

    public void Acelerar() { ... }
}
Objeto (Instancia)
Coche miCoche = new Coche();
miCoche.Color = "Rojo";
miCoche.Velocidad = 0;

Identificadores

Son los nombres que se asignan a los elementos de programación, como clases, métodos y variables. Deben comenzar con una letra o un guion bajo, y no pueden contener espacios. C# es sensible a mayúsculas y minúsculas.

Ejemplos: `miVariable`, `_nombre`, `NombreDeUsuario`.

string miVariable;

Palabras Clave

Son palabras reservadas por el lenguaje que tienen un significado especial y no pueden ser usadas como identificadores. Son la base de la sintaxis y las estructuras de C#.

Ejemplos: `class`, `public`, `private`, `int`, `string`, `if`, `for`.

public class Persona { ... }

Variables y Tipos de Datos

Las variables son espacios de memoria para almacenar datos. Cada una debe tener un tipo de dato que define su valor. Haz clic en los tipos para ver más.

Tipos Numéricos

Enteros:

  • **`int`**: Números enteros (e.g., `10`).
  • **`long`**: Números enteros grandes.
  • **`short`**: Números enteros pequeños.

Punto Flotante:

  • **`float`**: Números con decimales de precisión simple (e.g., `10.5f`).
  • **`double`**: Números con decimales de precisión doble (e.g., `10.5`). Es el tipo por defecto para decimales.
  • **`decimal`**: Alta precisión para cálculos financieros.
int edad = 20;
double salario = 50000.75;
Tipos de Texto

`string`:

Una secuencia de caracteres. Se usa para almacenar texto. Se declara entre comillas dobles.

string nombre = "Juan Pérez";
Tipo Booleano

`bool`:

Almacena un valor de verdad. Solo puede ser `true` o `false`.

bool esAdmin = true;

Blazor: Desarrollo Web con C#

Blazor es la arquitectura web que nos permite usar C# para crear interfaces de usuario interactivas. Se basa en componentes reutilizables, permitiendo que la lógica se ejecute tanto en el cliente como en el servidor.

  • **Componentes (Razor):** Los componentes son la base. Son bloques de interfaz de usuario reutilizables que combinan HTML y C#.
  • **Lógica de Negocio (C#):** Toda la lógica y la manipulación de datos se manejan con C#, eliminando la necesidad de JavaScript para la mayoría de las tareas.
  • **Modos de Renderizado (Server/WebAssembly):** La lógica se puede ejecutar en el servidor (Blazor Server) para una carga inicial más rápida o directamente en el navegador (Blazor WebAssembly) para una experiencia sin servidor.

El Lenguaje de Blazor: Razor

Razor es la sintaxis que une HTML y C#. El símbolo `@` es la clave para integrar la lógica de C# directamente en el marcado HTML, haciendo el código más limpio y productivo.

<h1>¡Hola, @nombreUsuario!</h1>

Blazor en el Mercado Laboral

Ventajas de Blazor

  • **Un solo lenguaje:** Permite a los desarrolladores de .NET usar C# para la lógica tanto del backend como del frontend, reduciendo la complejidad.
  • **Rendimiento:** Blazor Server es conocido por su alto rendimiento, ideal para aplicaciones empresariales.
  • **Ecosistema .NET:** Se integra de forma nativa con todo el ecosistema de Microsoft (ASP.NET Core, Entity Framework, Azure), facilitando el desarrollo de soluciones completas.
  • **Código reutilizable:** La arquitectura de componentes de Razor fomenta la reutilización de código y la creación de interfaces de usuario modulares.

Desventajas de Blazor

  • **Tamaño de descarga inicial:** Las aplicaciones Blazor WebAssembly pueden tener un tamaño inicial mayor, lo que podría afectar los tiempos de carga en conexiones lentas.
  • **Curva de aprendizaje:** Aunque es intuitivo para los desarrolladores de C#, puede representar una curva de aprendizaje para aquellos nuevos en el ecosistema .NET.
  • **Dependencia de Microsoft:** La tecnología está fuertemente ligada a Microsoft, lo que podría ser una desventaja para proyectos que buscan una independencia total de proveedores.

Uso y Demanda

Blazor ha ganado tracción rápidamente, especialmente en el desarrollo de aplicaciones empresariales, dashboards y sistemas internos. La demanda de desarrolladores Blazor está creciendo, y los salarios suelen ser competitivos debido a la especialización requerida y a la importancia de los proyectos en los que se utiliza.

Se utiliza comúnmente en entornos donde la infraestructura de .NET ya está presente, como en la banca, la salud y la manufactura, donde se valora el rendimiento, la seguridad y el robusto soporte de Microsoft.

Práctica de la Sesión 1: ¡Manos a la Obra con Blazor!

Sigue estos pasos para crear tu primera aplicación web interactiva con Blazor. ¡No te preocupes si no lo entiendes todo a la primera, lo importante es practicar!

  1. Paso 1: Crear la Aplicación

    Inicia Visual Studio y crea un nuevo proyecto. Elige la plantilla **Blazor Web App**.

  2. Paso 2: Explorar el Proyecto

    Familiarízate con la estructura de carpetas. Haz clic en cada una para saber qué contienen.

    📁 wwwroot

    📁 Components

    📁 Pages

    `wwwroot` (Archivos estáticos):

    Contiene todos los archivos públicos de tu aplicación web, como hojas de estilo CSS, imágenes, archivos JavaScript y otros activos estáticos. Es el equivalente a la carpeta "public" en otros frameworks.

    `Components` (Tus componentes):

    Aquí se almacenan los componentes de la interfaz de usuario de Blazor que puedes reutilizar en tu aplicación. Contiene subcarpetas como `Layout`, que define el diseño de la página, y `Pages`.

    `Pages` (Páginas de la aplicación):

    Esta carpeta es una subcarpeta de `Components` y contiene los componentes de Blazor que actúan como páginas completas, accesibles a través de una URL. Aquí es donde creamos el componente `Saludo.razor`.

  3. Paso 3: Crear el Componente de Saludo

    Dentro de la carpeta `Components/Pages`, crea un nuevo archivo llamado **`Saludo.razor`**.

    @page "/saludo"


    ¡Hola Mundo!
    

    @elMensaje
    

@code {
    private string elMensaje = "¡Hola!";
}

  4. Paso 4: Hacerlo Interactivo

    Añade un campo de texto y un botón para que el usuario pueda ingresar su nombre y el saludo se personalice.

    @page "/saludo"


    ¡Bienvenido a Blazor!
    

    @elMensaje

    private string? Nombre;

    private void Saludar() {
        elMensaje = "¡Hola " + Nombre + "!";
    }
